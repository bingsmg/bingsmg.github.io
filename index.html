记录第一次部署Hexo博客

发表于 2022-11-04

Hexo 搭建博客

https://hexo.io/zh-cn/

hexo 是一个快速、简洁且高效的博客框架。

部署前准备：git、node.js

注意一切按照官网流程走就得了，不容易踩坑，因为版本更新都有说明。

配置

https://hexo.io/zh-cn/docs/configuration

根据官网来就完事，可定制化选项都有。

需要注意的一点就是部署的问题：

通过安装 git 插件一键部署需要注意几个点：

  • 首先你得安装一键部署插件
  • 其次你得在本地配置 ssh 并把公钥拷贝到 github 里
  • 注意提示，github 自 2021 年起需要 token 验证
  • 配置的仓库地址需要用 ssh 地址，别用 https【避免每次输入密码】

主题

https://hexo.io/themes/

个人使用的是 NexT，官网：https://theme-next.js.org/docs/getting-started/

要自定义具体配置，按照官网介绍来就完事。当然可以去网上找别人自定义好的优秀的主题。

加速

看到一个特别好的博客：

使用 cdn 加速 Github Pages 的访问速度

总得来说呢，加速网页反应速度有两点：

  1. 网页懒加载，动态加载，也就是优先加载生成网页框架，再慢慢加载其他资源；
  2. CDN 内容分发网络，也就是基于缓存；

主流的 CDN 服务提供平台有阿里云、腾讯云、七牛云等等

以上大部分针对国内加速的 cdn 服务都需要网站备案才能使用，但是由于我的 Github Pages 是托管在 Github 上的，在国外，所以无法备案。

选择免费且不用备案的 CDN 服务提供平台 CloudFlare

加速流程：

  1. 注册账号
  2. 添加网站（需要加速的站点，不带 www），并选择免费计划
  3. 更改 DNS 服务器，并配置
